styled-ppx integrates with Melange projects through the standard opam and dune toolchain. The PPX transforms `[%styled.div ""]` and `[%cx ""]` extensions at compile time, and the `styled-ppx.melange` library provides the CSS runtime your compiled JavaScript output depends on. This guide walks through installation, both dune configuration patterns, and a complete Reason syntax example.

<Note>
  This guide assumes you already have a working Melange project set up with opam and dune. If you are starting from scratch, follow the [Melange getting started documentation](https://melange.re/v3.0.0/getting-started.html) first.
</Note>

<Steps>
  <Step title="Install via opam">
    Install `styled-ppx` from the opam package registry.

    ```bash
    opam install styled-ppx
    ```

    This installs two components:

    | Package | Purpose |
    |---|---|
    | `styled-ppx` | The PPX binary — transforms `[%styled.div ""]`, `[%cx ""]`, and `[%css ""]` at compile time |
    | `styled-ppx.melange` | The runtime library — exposes the `CSS` module in your compiled JavaScript output |
  </Step>

  <Step title="Configure dune">
    You need to add `styled-ppx` to the `(pps ...)` preprocessor list and `styled-ppx.melange` to the `(libraries ...)` list. There are two places this configuration can live depending on your project layout.

    **Option A — inside a `(library ...)` stanza with `(modes melange)`:**

    ```diff
    (library
      (name ...)
      (modes melange)
      (libraries
    +   styled-ppx.melange
        reason-react)
      (preprocess
        (pps
    +     styled-ppx
          melange.ppx
          reason-react-ppx)))
    ```

    **Option B — inside a `(melange.emit ...)` stanza:**

    ```diff
    (melange.emit
      (libraries
    +   styled-ppx.melange
        reason-react)
      (preprocess
        (pps
    +     styled-ppx
          melange.ppx
          reason-react-ppx)))
    ```

    <Note>
      `reason-react` and `reason-react-ppx` are optional. They are only required when using `[%styled.div]` to produce React components. If you only use `[%cx]` for class name generation, you can omit them.
    </Note>
  </Step>

  <Step title="Write your first styled component">
    With the dune configuration in place, you can use styled-ppx in any `.re` or `.ml` file in your library. The example below uses Reason syntax to define a styled anchor component with a dynamic `color` prop and a layout class name.

    <CodeGroup>
      ```reason Reason
      /* This is a ReasonReact module with those styles encoded as a unique className */
      module Link = [%styled.a (~color=CSS.hex("4299E1")) => {|
        font-size: 1.875rem;
        line-height: 1.5;
        text-decoration: none;
        margin: 0px;
        padding: 10px 0px;
        color: $(color);
      |}];

      /* This is a unique className generated by the styles */
      let layout = [%cx {|
        display: flex;
        width: 100%;
        height: 100%;
        justify-content: center;
        align-items: center
      |}];

      /* Later in a component */
      <div className=layout>
        <Link
          color={CSS.hex("333333")}
          href="https://sancho.dev"
          rel="noopener noreferrer">
          {React.string("sancho.dev")}
        </Link>
      </div>;
      ```

      ```rescript ReScript
      module Link = %styled.a((~color=CSS.hex("4299E1")) => `
        font-size: 1.875rem;
        line-height: 1.5;
        text-decoration: none;
        margin: 0px;
        padding: 10px 0px;
        color: $(color);
      `)

      /* This is a unique className generated by the styles */
      let layout = %cx(`
        display: flex;
        width: 100%;
        height: 100%;
        justify-content: center;
        align-items: center
      `)

      /* Later in a component */
      <div className=layout>
        <Link
          color={CSS.hex("333333")}
          href="https://sancho.dev"
          rel="noopener noreferrer">
          {React.string("sancho.dev")}
        </Link>
      </div>
      ```
    </CodeGroup>

    The PPX hashes every CSS block at compile time and generates a unique, stable class name string. The corresponding CSS rules are extracted to a static stylesheet at build time — no runtime CSS parsing is needed.
  </Step>
</Steps>

## Available extensions

| Extension | Description |
|---|---|
| `[%styled.div]`, `[%styled.a]`, etc. | Creates a typed React component wrapping the given HTML element with the supplied styles |
| `[%cx]` | Generates a single unique `className` string from a block of CSS |
| `[%css]` | Generates a list of `Css.rule` values for lower-level composition |
