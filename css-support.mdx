styled-ppx parses CSS according to [CSS Syntax Module Level 3](https://www.w3.org/TR/css-syntax-3/) and translates it into type-safe OCaml/ReScript bindings at compile time. Coverage varies by CSS module — some are fully supported, others partially, and a few are not yet implemented. This page documents exactly what you can rely on today.

## Levels of Support

There are three distinct layers to how styled-ppx handles CSS, and a given feature can be at any of these levels:

- **Parser support** — The CSS parser recognises and accepts the syntax. If a property or value fails at this level, styled-ppx will emit a compile-time error.
- **Code generation support** — The parsed CSS can be transformed into typed OCaml/ReScript bindings that are emitted as output. This is where type-checking happens.
- **Interpolation support** — Values from your OCaml/ReScript code can be spliced into CSS expressions as type-safe holes, ensuring that the value's type matches what the CSS property expects.

<Note>
  Not every property that parses will also have full code generation or interpolation support. When in doubt, refer to the [test suite on GitHub](https://github.com/davesnx/styled-ppx/blob/main/packages/ppx/test/css-support) for the most up-to-date picture of what works end-to-end.
</Note>

## Legacy CSS Specifications

The following baseline specifications are fully supported (✅) and underpin everything else:

- [(CSS 1.0) Cascading Style Sheets Level 1](https://www.w3.org/TR/CSS1/)
- [(CSS 2.1) Cascading Style Sheets Level 2 Revision 1 Specification](https://www.w3.org/TR/CSS21/)
- [(CSS 2.2) Cascading Style Sheets Level 2 Revision 2 Specification](https://www.w3.org/TR/CSS22/)
- [(CSS 3.0) CSS Syntax Module Level 3](https://www.w3.org/TR/css-syntax-3/)
