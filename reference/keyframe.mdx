CSS animations require two things: a set of keyframes that describe how styles change over time, and a reference to those keyframes in an `animation-name` declaration. styled-ppx provides the `%keyframe` extension to handle both concerns in a fully type-safe way — the extension validates the keyframe syntax at compile time and returns an opaque value that can only be used where an animation name is expected.

## Defining and using a keyframe

Declare keyframes with `%keyframe` and then interpolate the result directly into an `animation-name` property inside any `%styled` or `%cx` block.

<CodeGroup>
```rescript ReScript
let fadeIn = %keyframe(`
  0% {
    opacity: 0;
  }

  100% {
    opacity: 1;
  }
`)

module Component = %styled.div(`
  animation-name: $(fadeIn);
  width: 100px;
  height: 100px;
`)
```
```reason Reason
let fadeIn = [%keyframe {|
  0% {
    opacity: 0;
  }

  100% {
    opacity: 1;
  }
|}];

module Component = [%styled.div {|
  animation-name: $(fadeIn);
  width: 100px;
  height: 100px;
|}];
```
</CodeGroup>

## Optional outer braces

The outer curly braces surrounding the keyframe steps are optional. Both forms below are equivalent:

<CodeGroup>
```rescript ReScript
// With braces
let spin = %keyframe(`{
  from { transform: rotate(0deg); }
  to   { transform: rotate(360deg); }
}`)

// Without braces — identical output
let spin = %keyframe(`
  from { transform: rotate(0deg); }
  to   { transform: rotate(360deg); }
`)
```
```reason Reason
(* With braces *)
let spin = [%keyframe {| {
  from { transform: rotate(0deg); }
  to   { transform: rotate(360deg); }
} |}];

(* Without braces — identical output *)
let spin = [%keyframe {|
  from { transform: rotate(0deg); }
  to   { transform: rotate(360deg); }
|}];
```
</CodeGroup>

## Opaque return type

`%keyframe` returns an **opaque type**. Its internal representation is a `string` (a hashed name, generated the same way classnames are), but the type system prevents it from being used as an arbitrary string. You can only pass it where an animation name is accepted — for example, via `$(fadeIn)` interpolation inside `animation-name`.

<Note>
  Because the keyframe name is derived from a hash of its content, two identical keyframe definitions will resolve to the same name and will be de-duplicated in the emitted CSS — just like classnames generated by `%cx`.
</Note>
