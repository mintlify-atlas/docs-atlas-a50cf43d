Selectors let you style child elements, pseudo-classes, pseudo-elements, and more — all from within a single `%cx` or `%styled` block. The syntax is intentionally close to SASS, with one important rule: every nested selector must begin with `&` to make the relationship explicit.

## Nesting with `&`

The `&` token represents the generated classname. Any rule nested inside a block must be prefixed with `&`, whether you are targeting a pseudo-class, a pseudo-element, a child element, or an attribute selector.

<CodeGroup>
```rescript ReScript
let link = %cx(`
  color: $(Color.Text.body);

  &:hover {
    color: $(Color.Text.accent);
  }
`)
```
```reason Reason
let link = [%cx {|
  color: $(Color.Text.body);

  &:hover {
    color: $(Color.Text.accent);
  }
|}];
```
</CodeGroup>

<Note>
  Selectors nested inside a block are scoped to the classname generated by `%cx` or `%styled`. They are not global — they will not accidentally affect other elements on the page.
</Note>

## Supported selector types

styled-ppx validates selectors against **CSS Selectors Level 3** and the stable subset of **Level 4** (currently in draft). The following selector categories are all supported:

| Category | Examples |
|---|---|
| Type selectors | `& span`, `& p` |
| Class selectors | `& .active`, `& .$(myClass)` |
| ID selectors | `& #header` |
| Attribute selectors | `& [disabled]`, `& [type="text"]` |
| Pseudo-classes | `&:hover`, `&:focus`, `&:nth-child(2)` |
| Pseudo-elements | `&::before`, `&::placeholder` |
| Combinators | `& > span`, `& + p`, `& ~ li` |
| Comma lists | `&:hover, &:focus` |

## One level of nesting

Only **one level** of selector nesting is supported. Deeper nesting may compile without an error but will produce incorrect CSS output.

<Warning>
  Nesting selectors more than one level deep (e.g., `& .parent { & .child { ... } }`) is not supported. Move deeply nested rules to a flat `&`-prefixed selector or split them into separate `%cx` / `%styled` calls.
</Warning>

## Interpolation in selectors

You can interpolate any string expression inside a selector using `$(expr)`. This is especially useful for referencing classnames generated by other `%cx` calls so you can compose styles without hard-coding strings.

<CodeGroup>
```rescript ReScript
// `outlined` is a classname generated by `%cx`, but it can be any string
let outlined = %cx(`
  padding: 4px;
  outline: 1px solid red;
`)

let link = %cx(`
  color: $(Color.Text.body);

  & .$(outlined) {
    padding: 0px;
  }
`)

<div className={link ++ " " ++ (isActive ? outlined : "")} />
```
```reason Reason
// `outlined` is a classname generated by `[%cx]`, but it can be any string
let outlined = [%cx {|
  padding: 4px;
  outline: 1px solid red;
|}];

let link = [%cx {|
  color: $(Color.Text.body);

  & .$(outlined) {
    padding: 0px;
  }
|}];

<div className={(link ++ " ") ++ (if (isActive) {outlined} else {""})} />;
```
</CodeGroup>

<Tip>
  Interpolating a classname in a selector (e.g., `& .$(outlined)`) lets you conditionally apply a sibling style class while keeping both definitions fully type-checked at compile time.
</Tip>

## Selectors and `%css`

The `%css` extension generates a single `Css.rule` and therefore **does not support selectors or nesting**. If you need selectors, use `%cx` or a `%styled` component instead.
