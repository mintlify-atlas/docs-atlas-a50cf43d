The `CSS` module is styled-ppx's runtime library, available since [v0.40](https://github.com/davesnx/styled-ppx/releases/tag/0.40.0). It exposes every CSS value (like `CSS.px(10)` or `CSS.block`) and every CSS property (like `CSS.opacity` or `CSS.display`) as typed OCaml functions. The PPX transformation uses this module under the hood when it expands `%cx`, `%css`, and `%styled.tag` calls—but you can also import and use it directly to create values you then interpolate into those extensions.

## Package Names by Platform

The `CSS` module ships under different package names depending on your target platform.

<Tabs>
  <Tab title="ReScript">

    ```json
    "@davesnx/styled-ppx/rescript"
    ```

    `@davesnx/styled-ppx/rescript` is the library that exposes the `CSS` module with all runtime values and property functions.
  </Tab>
  <Tab title="Melange">

  </Tab>

## Values

Every CSS value is available as a typed OCaml function or constant in the `CSS` module. These are the building blocks you pass to CSS property functions or interpolate into extension strings.

<CodeGroup>
```rescript ReScript
let ten = CSS.px(10)
let position = CSS.block
let faded = CSS.rgba(0, 0, 0, #num(0.5))
```

```reason Reason
let ten = CSS.px(10);
let position = CSS.block;
let faded = CSS.rgba(0, 0, 0, `num(0.5));
```
</CodeGroup>

### Values as Polymorphic Variants

Every `CSS` module value is also available as a polymorphic variant. The two forms are fully interchangeable — they compile to the same type.

<CodeGroup>
```rescript ReScript
let ten = #px(10)      // same as CSS.px(10)
let pos = #block       // same as CSS.block
let col = #flex        // same as CSS.flexBox
```

```reason Reason
let ten = `px(10);     /* same as CSS.px(10) */
let pos = `block;      /* same as CSS.block */
let col = `flex;       /* same as CSS.flexBox */
```
</CodeGroup>

<Note>
  Use whichever form you prefer. The `CSS` module aliases exist for discoverability. Polymorphic variants are idiomatic OCaml/Reason and may be more concise in complex expressions.
</Note>

### Length Values

The `length` type encompasses every CSS length unit:

| Constructor | Description | Example |
|---|---|---|
| `CSS.px(n)` | Pixels (integer) | `CSS.px(16)` |
| `CSS.pxFloat(n)` | Pixels (float) | `CSS.pxFloat(1.5)` |
| `CSS.rem(n)` | Root em units | `CSS.rem(1.25)` |
| `CSS.em(n)` | Em units | `CSS.em(2.0)` |
| `CSS.vh(n)` | Viewport height | `CSS.vh(100.0)` |
| `CSS.vw(n)` | Viewport width | `CSS.vw(50.0)` |
| `CSS.pct(n)` | Percentage | `CSS.pct(100.0)` |
| `CSS.cm(n)` | Centimetres | `CSS.cm(2.54)` |
| `CSS.mm(n)` | Millimetres | `CSS.mm(10.0)` |
| `CSS.pt(n)` | Points | `CSS.pt(12)` |
| `CSS.zero` | Zero (unitless) | `CSS.zero` |

### CSS Variables

The `CSS.var` function creates a `var()` reference:

<CodeGroup>
```rescript ReScript
// Bare variable name (-- prefix is added automatically)
let gap = CSS.var("spacing-md")
// -> var(--spacing-md)

// With fallback
let color = CSS.var(~default="blue", "brand-color")
// -> var(--brand-color, blue)
```

```reason Reason
/* Bare variable name (-- prefix is added automatically) */
let gap = CSS.var("spacing-md");
/* -> var(--spacing-md) */

/* With fallback */
let color = CSS.var(~default="blue", "brand-color");
/* -> var(--brand-color, blue) */
```
</CodeGroup>

## Properties

Every CSS property is available as a camelCased function on the `CSS` module. Each function takes the correct typed value(s) and returns a `Css.rule` — the building block used by the [Array API](/reference/array).

<Tip>
  When writing plain `%cx` or `%styled.tag` strings, the PPX generates these property calls for you automatically. You only need to call them directly when building rules with the Array API.
</Tip>

### Simple Properties

Properties that accept a single value follow a straightforward pattern:

<CodeGroup>
```rescript ReScript
let faded     = CSS.opacity(0.9)
let invisible = CSS.visibility(#hidden)
let pointer   = CSS.cursor(#pointer)
let text      = CSS.color(CSS.hex("333333"))
let round     = CSS.borderRadius(CSS.px(8))
```

```reason Reason
let faded     = CSS.opacity(0.9);
let invisible = CSS.visibility(`hidden);
let pointer   = CSS.cursor(`pointer);
let text      = CSS.color(CSS.hex("333333"));
let round     = CSS.borderRadius(CSS.px(8));
```
</CodeGroup>

## `CSS.unsafe`

`CSS.unsafe` is the escape hatch for properties that styled-ppx does not yet support. It takes a raw property name (as written in CSS, not camelCase) and a raw string value, and returns a `Css.rule`.

**Signature:**

```ocaml
val unsafe : string -> string -> Css.rule
```

**When to use it:** When a property isn't in styled-ppx's built-in list yet. The safety guarantee disappears — if you write an invalid property or value, there will be no compile-time error.

<CodeGroup>
```rescript ReScript
let prefixed = CSS.unsafe("-webkit-tap-highlight-color", "transparent")

// Combine with the Array API
let card = %cx([
  %css("border-radius: 8px"),
  CSS.unsafe("-webkit-overflow-scrolling", "touch"),
])
```

```reason Reason
let prefixed = CSS.unsafe("-webkit-tap-highlight-color", "transparent");

/* Combine with the Array API */
let card = [%cx [|
  [%css "border-radius: 8px"],
  CSS.unsafe("-webkit-overflow-scrolling", "touch"),
|]];
```
</CodeGroup>

<Note>
  If a property you need is missing, please [open an issue](https://github.com/davesnx/styled-ppx/issues/new) — most properties are straightforward to add. `CSS.unsafe` is a temporary workaround, not a design goal.
</Note>

## SSR / Native APIs

When targeting OCaml native (using `styled-ppx.native`), the `CSS` module exposes additional server-side rendering utilities. These are **not** available in the ReScript or Melange targets.

### `CSS.get_stylesheet()`

Returns a `string` containing all CSS rules that have been generated so far. Use this when you want to inline the stylesheet as a raw string — for example, when writing it to a file or injecting it into a custom `<style>` tag.

```ocaml
val get_stylesheet : unit -> string
```

```reason
let className = [%cx {|
  display: flex;
  align-items: center;
|}];

let stylesheet: string = CSS.get_stylesheet();
/* ".css-1xuw4bg { display: flex; align-items: center; }" */
```

### `CSS.style_tag`

A server-reason-react component that renders a `<style>` element containing all generated CSS. Place it inside your document `<head>` to ship styles as part of server-rendered HTML.

```reason
module Page = {
  [@react.component]
  let make = () => {
    <html>
      <head>
        <CSS.style_tag />
      </head>
      <body>
        /* ... */
      </body>
    </html>
  };
};
```

<Note>
  Because `CSS.style_tag` collects styles that have already been executed, you must render your application components *before* rendering the document shell that contains `<CSS.style_tag />`. A safe pattern is to call `ReactDOM.renderToString(<App />)` first, then render the outer `<Document />` that references `<CSS.style_tag />`.
</Note>

### `CSS.get_string_style_hashes()`

Returns a space-separated string of all generated class-name hashes. This is used when building the `data-emotion` attribute required for client-side hydration.

```ocaml
val get_string_style_hashes : unit -> string
```

To support React hydration, construct the `<style>` tag manually with the correct emotion attributes:

```reason
React.createElement("style",
  [
    Bool("data-s", true),
    String("data-emotion", "css " ++ CSS.get_string_style_hashes()),
    DangerouslyInnerHtml(CSS.get_stylesheet())
  ], []
)
```

### SSR API Summary

| Function | Return type | Use case |
|---|---|---|
| `CSS.get_stylesheet()` | `string` | Inline all CSS as a raw string |
| `CSS.style_tag` | React element | Drop-in `<style>` tag for server-rendered HTML |
| `CSS.get_string_style_hashes()` | `string` | Build the `data-emotion` attribute for hydration |

## Usage with Interpolation

Values created with the `CSS` module are designed to be interpolated inside `%cx`, `%css`, and `%styled.tag` strings using the `$(variable)` syntax. The PPX uses the interpolation position to infer the expected type, so passing a value of the wrong CSS type produces a compile error.

<CodeGroup>
```rescript ReScript
let small = CSS.px(10)
// CSS.length — valid wherever a length is expected

let classname = %cx(`margin: $(small)`)
// -> margin: 10px;

module Theme = {
  let black = CSS.hex("111111")
}

module Link = %styled.a(`
  color: $(Theme.black);
  font-size: $(small);
`)
```

```reason Reason
let small = CSS.px(10);
/* CSS.length — valid wherever a length is expected */

let classname = [%cx {|margin: $(small)|}];
/* -> margin: 10px; */

module Theme = {
  let black = CSS.hex("111111");
};

module Link = [%styled.a {|
  color: $(Theme.black);
  font-size: $(small);
|}];
```
</CodeGroup>

See the [Interpolation](/reference/interpolation) reference page for a full explanation of rules, supported positions, and type-safety guarantees.
