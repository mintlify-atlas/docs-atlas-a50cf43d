styled-ppx is a [PPX](https://tarides.com/blog/2019-05-09-an-introduction-to-ocaml-ppx-ecosystem) (preprocessor extension) and runtime library that brings styled components to ReScript, Melange, and OCaml. Write CSS directly in your source code and get type errors at compile time—not runtime surprises in the browser.

## What is styled-ppx?

A PPX is OCaml's macro system—a small program that runs before compilation and transforms your source code. styled-ppx uses this to expand CSS extension nodes like `%styled.div`, `%cx`, `%css`, and `%keyframe` into fully typed, validated code backed by its own CSS runtime.

```rescript
module Center = %styled.div(`
  height: 100vh;
  width: 100vw;
  display: flex;
  align-items: center;
  justify-content: center;
`)

<Center> {React.string("Hello from the future!")} </Center>
```

```reason
module Center = [%styled.div {|
  height: 100vh;
  width: 100vw;
  display: flex;
  align-items: center;
  justify-content: center;
|}];

<Center> {React.string("Hello from the future!")} </Center>;
```

## How It Works

styled-ppx operates entirely at compile time. The PPX reads the CSS string you write, parses it against CSS Syntax Module Level 3, type-checks every property and value, hashes the resulting rules into unique class names, and emits OCaml code that calls the CSS runtime to attach styles at load time.

<Steps>
  <Step title="You write CSS in an extension node">
    ```rescript
    let layout = %cx(`display: flex; align-items: center;`)
    ```
  </Step>
  <Step title="The PPX parses and type-checks your CSS">
    If a property or value is invalid, you get a compile error pointing to the exact location.
  </Step>
  <Step title="The PPX emits typed OCaml code">
    The extension expands to a call like `CSS.make("css-<hash>-layout", [])` — the class name string and CSS text are both computed at compile time and embedded directly in the output.
  </Step>
  <Step title="The runtime attaches styles at load time">
    The pre-computed CSS rules (stored as module-level attributes) are rendered into a `<style>` element and injected into the document head when the module loads.
  </Step>
</Steps>

## Extensions at a Glance

| Extension | ReScript | Reason/OCaml | Returns |
|-----------|----------|--------------|---------| 
| Styled component | `%styled.div(...)` | `[%styled.div ...]` | React component module |
| Class name | `%cx(...)` | `[%cx ...]` | `string` (className) |
| Single rule | `%css(...)` | `[%css ...]` | `Css.rule` |
| Keyframe | `%keyframe(...)` | `[%keyframe ...]` | Opaque keyframe ref |
| Global styles | `%styled.global(...)` | `[%styled.global ...]` | `unit` |
